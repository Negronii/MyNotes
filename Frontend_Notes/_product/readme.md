# `_product/`

Meta-level thinking for `Frontend_Notes/` — how we write, why we write it that way, and who it's for.

Content notes live in the numbered files at the `Frontend_Notes/` root (`1.1`, `2.6`, `4.2`, …). This folder holds the reasoning *behind* those notes: writing conventions, reader personas, and future project thoughts.

## Who this is for

- **The author**, to write consistently across many notes and many sittings.
- **Future contributors and AI assistants** helping with edits, so the style holds as the repo grows.
- **Curious readers**, to understand why the notes are shaped the way they are.

## Index

- [`writing-guide.md`](./writing-guide.md) — the mindset doc. Read this before writing or rewriting a note. Covers the three-type note taxonomy (Concept / Question-bundle / Reference), the Pyramid Principle adapted for interview prep, six writing principles tagged by type, three copy-paste templates, and a type-aware self-review checklist.

## Conventions

- Folder is prefixed with `_` so it sorts above numbered notes and signals "meta, not content".
- Docs here should be short, opinionated, and actionable. If a doc turns into a long essay, it probably belongs elsewhere.
- This folder is scoped to `Frontend_Notes/` for now. Other sections of the repo can adopt the guide if useful, but aren't required to.
