# `_product/`

Meta-level thinking for `AI_Engineering_Notes/` — how we write, why we write it that way, and who it's for.

Content notes live in the numbered files at the `AI_Engineering_Notes/` root (`1.1`, `2.1`, `4.1`, …). This folder holds the reasoning *behind* those notes: writing conventions, reader personas, and AI-specific sub-shapes to watch for.

## Who this is for

- **The author**, to write consistently across many notes and many sittings.
- **Future contributors and AI assistants** helping with edits, so the style holds as the repo grows.
- **Curious readers**, to understand why the notes are shaped the way they are.

## Index

- [`writing-guide.md`](./writing-guide.md) — the mindset doc. Read this before writing or rewriting a note. Adapts the [Frontend writing guide](../../Frontend_Notes/_product/writing-guide.md) for AI engineering: same three-type taxonomy (Concept / Question-bundle / Reference), same Pyramid Principle, plus AI-specific callouts for tradeoff questions, design-exercise sub-shape, framework/meta-process notes, and model-roster freshness.

## Conventions

- Folder is prefixed with `_` so it sorts above numbered notes and signals "meta, not content".
- Docs here should be short, opinionated, and actionable. If a doc turns into a long essay, it probably belongs elsewhere.
- This guide is a sibling of [`Frontend_Notes/_product/writing-guide.md`](../../Frontend_Notes/_product/writing-guide.md). The two share a spine; each has its own examples and domain-specific adaptations. If you change one, consider whether the sibling needs the same change.
